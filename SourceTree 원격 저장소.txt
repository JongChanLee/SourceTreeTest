

[Github을 기준으로 작성]

1.github.com에 로그인해서 저장소 새로 하나 만들기.
2. 만들고 나면 나오는 HTTPS 주소를 복사
3. 메뉴의 [Repository] - [add Remote]
4. [add] 버튼을 누르고 [URL / Path] 항목에 HTTPS를 붙여넣기 한다.
5. Remote name은 디폴트로 해도 되고 지정해도 된다.
6. Ok를 해서 원격 저장소를 생성한다.
7. 브랜치 목록이 있는 쪽 밑에 Remote에 원격저장소 목록이 생성된다.
8. 메뉴의 [Push] 를 선택하면 로컬 저장소(내 컴퓨터)에서 원격저장소(github)으로 백업을 할 수 있다.
9. 백업할 브랜치 (오픈 소스로 나가기 때문에 원하지 않는 것은 백업하지 않아도 된다.)를 선택하여 백업을 진행한다.
  -> 이 과정에서 github 아이디와 비밀번호를 입력하라는 창이 나온다.
  -> 녹색이 나오면 정상적으로 백업이 된것
-> commit 목록에 원격저장소 이름과 브랜치가 나온다.
  -> 원격저장소에 백업이 된 부분을 확인 할 수 있다.

-> github에 백업을 할 경우 로컬 저장소에서 commit 한 내용을 github에서 그대로 볼 수 있다.


원격 저장소에 백업을 진행한 후에 로컬 저장소에서 버전 등록을 새로 했을 경우,

-> Push 아이콘에 숫자가 생긴다.
   -> 원격 저장소와 로컬 저장소에 버전 차이가 그 숫자만큼 존재 한다는 것을 의미한다.
   -> 버전 차이가 존재하는 브랜치 옆에도 버전 차이가 나는 만큼의 숫자가 나온다. 
-> 여기서 다시 push를 누르고 ok를 하면 업로드가 된다. (백업)


----------------------------------------------

[Clone] -> 다른 사용자가 원격 저장소를 자신의 로컬 저장소에 복제하는 것

1. 저장소 생성 버튼 [New/ Clone]을 눌러 [Clone Repository] 탭에서 작업을 한다.
2. 복제할 원격 저장소의 HTTPS의 주소를 복사 해서 Path / URL 란에 붙여넣는다.
3. Destination Path 란에는 복제하는 원격 저장소를 저장할 로컬 저장소의 주소를 집어넣는다.
4. OK를 한다.

***********협업****************
사용자 '가', '나'
협업 프로젝트 A 라고 하자

작업을 시작하기 전에 원격 저장소에서 업로드를 미리 해야한다. (충돌을 최대한 방지 하기 위해서)
[Pull]을 선택하여 가져온다.
원격 저장소에서 Pull을 하지 않고 Push를 하려고 하면 오류가 나서 Push를 할 수 없다.

Pull을 먼저 하지 않고, 수정을 하여 commit을 한 다음에
Pull을 하면 자동으로 Merge가 된고 commit까지 된다.

--- -- -- -- - - -- - -

가와 나가 협업을 동시에 하고 있을 때.
'가'가
Pull -> work(수정) -> commit 을 하고 Push를 하기 전에
  Pull을 한번 더 해준다.
    -> 그 이유는 저 3단계를 거치는 동안 '나'가 다른 것을 Push 하였을 수도 있기 때문이다.
때문에 '가'는
Pull -> work(수정) -> commit -> Pull을 한다. (이때 마지막 Pull에서 자동으로 commit을 해준다. 충돌이 없다면)
그 다음에 바로 Push를 하면 된다.
Pull -> work(수정) -> commit -> Pull -> Push
